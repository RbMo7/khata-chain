# Future Integrations — Recommended Build Order

## Priority 1 — Core gaps (blocking real usage)
1. **Partial payments** — touches existing repayment flow; most impactful
2. **Credit limit enforcement** — 1-field addition to store owner settings → enforced at credit creation
3. **UPI / Razorpay** — swap or add alongside Stripe for India-based payments
4. **WhatsApp / SMS reminders** — due date + overdue alerts via Twilio or WATI

## Priority 2 — Kirana-specific trust features
5. **Item-level ledger** — line items ("3kg atta, 1L oil") attached to a credit entry
6. **Guarantor / reference** — someone vouches for a new borrower with no reputation history
7. **Passbook / statement PDF export** — for disputes and record-keeping
8. **Multi-store reputation** — borrower score visible (with consent) across stores

## Priority 3 — Growth / analytics
9. **Store owner analytics dashboard** — at-risk borrowers, repayment rate, monthly collected vs outstanding
10. **Recurring / monthly tab** — open tab for the month, settle on the 1st
11. **Referral system** — existing borrower refers new one, gets reputation boost
12. **PWA / mobile install** — phone-first experience for kirana owners and customers
